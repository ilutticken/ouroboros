# Ouroboros Agent Rules

## Game Design & Clarity
1. **Telegraphing Mechanics**: Every new hazard, item, or mechanic must be clearly telegraphed to the player through UI text, narrative terminal hints, or visual indicators *before* they interact with it. Do not introduce silent rules that the player has to discover by dying unfairly.
2. **Total Diegesis**: Everything the player perceives — every sound, visual, HUD element, number, and menu — must be justifiable as a real signal happening *inside* the system fiction, never as feedback layered on from outside it. Before adding any output, answer "what in the world is producing this?"; if there is no in-world answer, do not ship it. A signal must not exist before its cause exists (no wall tone before walls, no corruption wubs before Glitches). See `design_doc.md` §2.5.
